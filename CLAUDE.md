# AI Dashboard 项目配置

## 产品调研与内容生成标准

### 📋 核心原则

本项目的所有内容生成（GitHub Trending、AI Products、Inspiration Library等）必须遵循**专业产品调研报告**的标准，确保内容的深度、专业性和可执行性。

---

## 🎯 六大核心模块框架

每次生成详细内容时，必须包含以下六大模块：

### 1. 调研背景与目标

在开始陈述发现之前，必须先明确这次调研的"初心"。

#### 必须包含：

**调研背景**：为什么要做这次调研？
- 是为了应对新入局的竞争者？
- 探索新的增长点？
- 还是为了解决现有产品的某个核心痛点（如留存率低）？

**核心目标**：你想通过这次调研回答什么问题？
- 目标必须具体
- ❌ 错误示例："了解竞品"
- ✅ 正确示例："验证我们即将上线的AI功能在交互体验上是否存在颠覆性方案"

**调研范围**：明确边界
- 只调研直接竞品，还是包含跨界标杆？
- 聚焦于某个特定功能模块，还是覆盖整个商业闭环？
- 调研时间范围和方法论

---

### 2. 市场与宏观环境

这部分用来判断"赛道"的现状和天花板。

#### 必须包含：

**PEST分析**：
- **政策（Political）**：相关法规、监管政策
- **经济（Economic）**：市场规模、增长趋势、投资环境
- **社会（Social）**：用户习惯变化、社会趋势
- **技术（Technology）**：技术成熟度、创新突破

**市场规模与趋势**：
- TAM（Total Addressable Market）：总潜在市场
- SAM（Serviceable Available Market）：可服务市场
- SOM（Serviceable Obtainable Market）：可获得市场
- 该品类处于导入期、成长期、成熟期还是衰退期？
- CAGR（复合年增长率）

**竞争格局**：
- 是寡头垄断、百花齐放，还是高度分散？
- 头部玩家占据了多少市场份额？
- 进入壁垒和退出壁垒
- 市场集中度（CR4、HHI指数）

---

### 3. 用户与需求洞察

产品调研的核心是"人"，而不仅仅是"物"。如果缺少这一环，调研就容易流于表面。

#### 必须包含：

**目标用户画像**（至少3-5个细分用户群）：
- 人口统计学特征：年龄、职业、收入、地域
- 心理特征：价值观、生活方式、态度
- 行为特征：使用场景、频率、偏好
- 付费能力和意愿
- 用户引用（User Quote）：真实用户的原话

**核心需求与痛点**：
- 用户在现有解决方案下，有哪些未被满足的期望？
- 是价格太贵、操作太复杂，还是售后服务跟不上？
- 痛点的严重程度排序（Must-have vs Nice-to-have）

**用户决策路径**（Decision Journey）：
- **认知阶段（Awareness）**：用户如何发现产品？
- **考虑阶段（Consideration）**：用户如何评估和对比？
- **购买阶段（Purchase）**：转化的关键因素和障碍
- **推荐阶段（Advocacy）**：用户如何成为布道者？
- 每个环节的转化率和流失率分析

**关键洞察（Key Insights）**：
- 3-5个反直觉或深度洞察
- 必须有数据或案例支撑
- 用"💡 洞察X："格式呈现

---

### 4. 竞品/标杆产品深度拆解

这是调研的主体部分，建议采用结构化框架，避免陷入"你有的功能我也有"的无意义罗列。

#### 筛选对标对象

**三类对标**：
1. **直接竞品**：解决同一类用户同一类需求
2. **间接竞品**：解决不同需求但抢占同一用户时间/预算
3. **参照品/标杆**：不同行业但用户体验或商业模式值得借鉴

#### 拆解维度

**战略层**：
- 产品的定位是什么？
- 核心卖点是"性价比"、"技术壁垒"还是"生态闭环"？
- 目标市场和用户群体
- 差异化策略

**范围层**：
- 核心功能有哪些？
- 哪些功能是"必须有"的基建？
- 哪些是"差异化"的亮点？
- 功能完整度对比

**结构层**：
- 信息架构是否清晰？
- 用户完成核心任务需要几步？
- 导航和路径设计

**框架层**：
- 交互逻辑是否顺畅？
- 关键按钮的引导是否明显？
- 反馈机制

**表现层**：
- 视觉风格、品牌调性如何？
- UI/UX质量
- 设计语言一致性

**数据表现**：
- 下载量、活跃用户数、付费率
- 用户评分及评价关键词
- **重要**：用户吐槽的点往往是机会点

**商业模式**：
- 定价策略（Freemium、订阅、按量付费）
- 收入结构（ToC、ToB、混合）
- 获客成本（CAC）和生命周期价值（LTV）
- 盈利能力和现金流

**对比矩阵**：
- 制作表格，横向对比3-5个竞品
- 至少8个维度的对比
- 用星级评分（⭐⭐⭐⭐⭐）直观呈现

---

### 5. 核心发现与差异化分析

单纯"拆解"只是信息搬运，"对比"才能产生洞察。

#### 必须包含：

**SWOT分析**（针对调研对象）：
- **Strengths（优势）**：3-6点，具体说明
- **Weaknesses（劣势）**：3-6点，不回避问题
- **Opportunities（机会）**：3-6点，趋势和空白
- **Threats（威胁）**：3-6点，风险和挑战

**关键发现（Key Findings）**：
- 3-5个核心发现
- 每个发现必须包含：
  - **标题**：一句话概括
  - **详细说明**：200-300字深度分析
  - **影响程度**：高/中/低影响，说明对决策的影响

**差异化分析**：
- **为什么赢（Why Winning）**：成功的关键因素（3-5点）
- **竞争护城河（Competitive Moat）**：可持续的竞争优势
- **风险对冲（Risk Hedge）**：如果竞争对手做出反应，应对策略是什么？

**反共识洞察**：
- 有没有发现一些数据或现象与行业直觉相悖？
- 这类洞察往往是产品创新的源泉
- 用"⚡ 反共识："标注

---

### 6. 结论与行动建议

这是调研报告价值转化的关键。如果前面分析了那么多，最后结论只是"知道了"，那调研就是失败的。

#### 必须包含：

**结论**：
- 用1-2句话概括核心结论
- 必须明确、可执行
- ✅ 示例："竞品A虽然功能强大，但上手门槛过高，且忽略了中小客户的价格敏感度，这为我们留出了'轻量级、高性价比'的切入空间。"

**行动建议（Recommendations）**：

**Do（建议立即执行）**：
- 按优先级排序（P0、P1、P2）
- 每个建议包含：
  - **行动（Action）**：具体做什么
  - **原因（Why）**：为什么要做
  - **方法（How）**：如何执行
  - **负责人（Owner）**：谁来负责
  - **时间线（Timeline）**：何时完成
  - **资源需求（Resources）**：需要什么资源

**Don't（明确不做）**：
- 指出哪些领域是红海或不符合当前战略
- 说明为什么不做
- 提供替代方案

**Validate（需要验证）**：
- 通过MVP（最小可行性产品）或A/B测试来验证哪些假设？
- 实验设计：假设、方法、成本、决策标准

**优先级排序**：
- 使用RICE模型（Reach × Impact × Confidence / Effort）
- 或使用其他框架（ICE、Value vs Effort等）
- 制作优先级矩阵表格

**执行路线图（Execution Roadmap）**：
- 按季度或月度分解
- 明确里程碑（Milestones）
- 可衡量的目标（OKR/KPI）

## 📝 内容生成规范

### GitHub Trending 项目

每个项目的 `detailedAnalysis` 必须包含：

```javascript
{
  projectOverview: "200-300字，包含项目背景、核心价值、用户规模",
  coreFeatures: [
    "功能1：具体说明（不只是功能名）",
    "功能2：包含技术细节或数据支撑",
    "功能3：强调差异化优势",
    // 5-6个核心功能
  ],
  whyTrending: "200-300字深度分析，包含：\n" +
    "- 解决了什么痛点\n" +
    "- 市场时机为什么对\n" +
    "- 相比竞品的优势\n" +
    "- 社区和生态因素\n" +
    "- 数据支撑（star增长、用户反馈等）"
}
```

### AI Products 产品

每个产品的 `detailedResearch` 必须包含：

```javascript
{
  productOverview: "200-300字产品概述，包含定位、核心功能、用户规模、商业表现",
  pricingStrategy: {
    "免费版": "详细说明限制和价值",
    "Pro版($XX/月)": "功能列表和目标用户",
    "企业版": "企业功能和定价",
    // 清晰的定价梯度
  },
  competitiveAdvantage: [
    "优势1：具体说明，包含数据或案例",
    "优势2：对比竞品的差异化",
    "优势3：技术或商业模式的独特性",
    // 3-4个核心竞争优势
  ]
}
```

### Inspiration Library 灵感

每个灵感的 `completeResearch` 必须包含：

```javascript
{
  marketOpportunity: {
    size: "市场规模数据（TAM/SAM/SOM）+ CAGR",
    painPoints: [
      "● 痛点1：具体场景和严重程度",
      "● 痛点2：当前解决方案的不足",
      "● 痛点3：用户付费意愿分析",
      // 3-4个关键痛点
    ]
  },
  optimizationIdeas: [
    "💡 创新点1：【标题】—— 详细说明如何解决痛点",
    "💡 创新点2：【标题】—— 技术实现和差异化",
    "💡 创新点3：【标题】—— 用户体验创新",
    "💡 创新点4：【标题】—— 商业模式创新",
    // 4个创新点
  ],
  businessModel: {
    freemium: "免费版策略",
    pro: "付费版定价和价值",
    enterprise: "企业版方案（如适用）",
    target_revenue: "目标收入预测（3-5年）+ ARPU分析"
  }
}
```

### AI Insights 行业洞察

每个问题的回答必须包含：

```javascript
{
  question: "精准的问题（针对产品经理核心关注点）",
  answer: "800-1500字深度回答，必须包含：\n" +
    "1. 核心观点（开篇明义）\n" +
    "2. 详细分析（分点阐述，至少3-5点）\n" +
    "3. 真实案例（成功案例 + 失败案例对比）\n" +
    "4. 数据支撑（具体数字、比例、增长率）\n" +
    "5. 可执行建议（Do/Don't/Validate）\n" +
    "6. 反共识洞察（如适用）"
}
```

## ✅ 质量检查清单

生成内容前，必须确认：

### 深度检查
- [ ] 每个项目/产品至少500字详细分析
- [ ] 包含具体数据和数字（不能只有定性描述）
- [ ] 有真实案例或用户引用
- [ ] 分析有逻辑层次（从现象到本质）

### 完整性检查
- [ ] 六大模块是否齐全？
- [ ] 每个模块的子项是否完整？
- [ ] SWOT分析是否深入？
- [ ] 行动建议是否可执行？

### 专业性检查
- [ ] 使用专业术语和框架（PEST、SWOT、RICE、PMF等）
- [ ] 数据来源可靠（标注估算/公开数据/调研数据）
- [ ] 避免空洞表述（"很好"、"不错"、"优秀"等）
- [ ] 具体量化（"提升50%"而非"大幅提升"）

### 可读性检查
- [ ] 使用emoji和格式化增强可读性
- [ ] 关键信息用**粗体**或「」标注
- [ ] 适当使用列表和表格
- [ ] 段落长度适中（不超过200字/段）

## 🎯 特别强调

### 数据真实性
- 所有数据必须合理（不能编造夸张数字）
- 优先使用公开数据或行业报告
- 如果是估算，必须标注"估计"、"约"等
- 数字要有逻辑一致性（市场规模、用户数、收入等要匹配）

### 案例真实性
- 优先使用真实公司和产品案例
- 如果使用虚构案例，必须合理且具有代表性
- 成功案例和失败案例要平衡

### 洞察深度
- 不满足于表面现象，要挖掘本质原因
- 使用"为什么"追问至少3次
- 寻找反常识、反直觉的洞察
- 连接不同领域的知识（跨界洞察）

### 可执行性
- 所有建议必须具体可落地
- 明确责任人、时间线、资源需求
- 提供优先级排序
- 说明成功标准和衡量指标

## 📊 效果标准

### 优秀内容的特征
✅ 读完后有"原来如此"的恍然大悟感
✅ 有具体数据和案例支撑，可信度高
✅ 提供了可直接执行的行动建议
✅ 揭示了反常识或深层次的洞察
✅ 逻辑清晰，结构完整
✅ 专业术语使用准确，但不晦涩

### 需要改进的信号
❌ 内容浮于表面，缺乏深度
❌ 全是定性描述，没有数据支撑
❌ 只罗列功能，没有分析原因
❌ 结论模糊，没有可执行建议
❌ 逻辑跳跃，缺少论证过程
❌ 用词空洞，"很好"、"不错"等词汇过多

---

## 🔄 持续改进

### 内容池维护
- 每周检查内容池，确保案例和数据时效性
- 优先使用最新（6个月内）的案例和数据
- 过时的内容要及时更新或替换

### 用户反馈
- 收集用户对内容深度的反馈
- 分析哪些内容点击率高（受欢迎）
- 根据反馈调整内容重点

### 质量提升
- 每次生成后自我评分（1-10分）
- 对比优秀案例，找差距
- 持续学习最新的产品分析方法论
